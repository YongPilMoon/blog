## TCP / IP model
TCP(Transmission Controll Protocol) 프로토콜과 IP 프로토콜을 OSI 7계층 형식에 맞추어 간략화 시킨 모델
주소창에 www.google.com 입력시 다음과 같은 과정이 발생합니다.
<br/>

### Application Layer
- www.google.com으로 HTTP GET 요청
- DNS에서 도메인 주소를 통호 IP주소 획득
- 전송계층으로 데이터 전달
<br/>
### Transport Layer
- HTTP 메세지 패킷으로 분리해서 
- 분해된 데이터를 판별하기 위해 번호 부여
- 인터넷 계층으로 데이터 전달

<br/>
### Internet Layer & Network Interface
- 어디있는지 인터넷을 통해 찾아다니면서 이동
<br/>
### Transport Layer
- 도착한 패킷을 일련번호를 보고 조립
- Application Layer로 데이터 전달
<br/>
### Application Layer
- 전달받은 데이터를 서버에 전달

## TCP vs UDP(User Datagram Protocol)
### 연결 여부
<br/>
TCP는 연결 설정 및 해제 과정이 필요(three hands shaking)
<br/>
### 신뢰성
- TCP는 신뢰성 있는 전송을 지원
- 데이터 전송 중에 발생하는 오류를 검사하고 복구할 수 있으며, 데이터 전송 순서 보장
- UDP는 신뢰성이 낮으며, 데이터 손실이나 오류가 발생할 수 있음
<br/>
### 전송 속도
- TCP는 신뢰성을 보장하기 위해 패킷을 확인하고 재전송해야 하므로 전송 속도가 느림
 - UDP는 패킷을 확인하지 않고 빠르게 전송되기 때문에 전송 속도가 빠름
<br/>
### 헤더 크기
- TCP는 연결 설정 및 해제 과정 등을 위한 제어 정보를 포함해 헤더가 큼
- UDP는 제어 정보가 적어 헤더 크기가 작음
<br/>
### 사용 용도
- TCP는 파일 전송, 이메일 전송, 웹 페이지 로딩 등에 주로 사용
- UDP는 실시간 데이터 전송에 주로 사용(온라인 게임, 동영상 스트리밍, 음성 통화)

### three way handshaking(SYN, SYN ACK, ACK RECEIVED)
- TCP에서 사용되는 연결 설정 프로세스
- 클라이언트와 서버간에 안전한 연결을 설정하기 위해 사용
