여러 서비스를 하나의 git repository에서 관리

## 장점
- 여러서비스의 공통된 패키지 버전을 한곳에 관리할 수 있음
- lint, ts-config 등 공통 설정을 한 곳에서 관리할 수 있음
- 여러 서비스의 공통 수정 사항을 한 컷밋으로 작성할 수 있음

## 주의점

- dependency 중복

## yarn workspace


root workspace 설정
<CodeBlock>
  {`
  "workspaces": [
    "packages/*"
  ],
`}
</CodeBlock>

root workspace 패키지 설치
`yarn add -W @emotion/react`
