## 13장 스코프
<br/>
### 스코프란?
**식별자**(변수 이름, 함수이름, 클레스 이름 등)가 `다른 코드에 의해 참조 될 수 있는 범위` 로 선언된 위치에 따라 결정된다.
<br/>
### 식별자란?
- 값을 식별해 낼 수 있는 고유한 이름
- 다른 스코프 내에는 같은 식별자를 사용할 수 있다.
<br/>
### 전역변수와 지역변수
- **전역 변수**는 전역 스코프에 선언된 변수로 `코드 모든 곳에서 접근 가능`하다.
- **지역 변수**는 함수 내에 선언된 변수로 `함수 내부와 하위 함수에서 접근 가능`하다.
<br/>
### 스코프 체인과 렉시컬 환경
- 스코프가 계층적으로 연결된 것을 **스코프 체인**이라 한다.
- 변수 참조 시 스코프 체인을 따라 `하위 스코프 부터 시작하여 상위 스코프로 이동하며` 검색한다.
- 스코프 체인은 렉시컬 환경에 의해 만들어진다.
- 렉시컬 환경은 `OuterEnvironmentReference`에 상위 스코프 참조 값을 저장한다.
<br/>
### 동적 스코프와 렉시컬 스코프
- **동적 스코프**는 `함수 호출 위치` **렉시컬 스코프**는 `함수 정의 위치`에 따라 상위 스코프가 결정된다.
- 자바스크립트는 렉시컬 스코프를 따른다.
- 함수는 코드 평가단계에서 자신이 정의된 스코프를 기억하고 호출 시 자신이 정의된 스코프를 상위 스코프로 사용한다.
<br/>

## 23장 실행 컨텍스트
<br/>
### 소스코드의 평가와 실행
- 자바스크립트 엔진은 소스 코드를 `평가`와 `실행`이라는 2단계로 나누어서 처리한다.
- **평가 단계**에서는 변수, 함수가 선언되어 레시컬 환경에 등록된다.
- **실행 단계**에서는 선언문을 제외한 소스코드가 실행된다.
<br/>
