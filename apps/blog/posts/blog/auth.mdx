HTTPS:  TLS를 통한 암호화

### Set-Cookie 설정

## HttpOnly => 쿠키의 자바스크립트 접근을 막는다.
크로스 사이트 스크립팅(XSS) => 자바스크립트 코드 injection
해커가 글을 포스팅 할 때 자바스크립트 코드를 injection 하고 사용자 글을 읽을 때 스크립트가 실행되면서 쿠키 혹은 localstorage 담긴 
인증정보를 탈취할 수 있다. 

## Secure: Https로만 쿠키가 전송되게 한다.

## HTTPS
대칭키 vs 비대칭키

대칭키: 메세지를 보내는 쪽과 메세지를 받는쪽이 똑같은 키를 가지고 암호화 복호화 한다.
비대칭키: 두개의 키가 사용(공개키와 개인키) => 서버가 개인키를 가지고 있고 클라이언트가 공개키를 가지고 암호화 해서 서버에 전송
서버만 개인키를 가지고 있음으로 서버만 내용을 복호화 할 수 있다.

handshaking 과정에서 인증서를 보냄 -> 브라우저에 내장된 CA(Certificate Authority)의 정보를 통해 진위여부 확인
-> 클라이언트는 handshaking때 생성된 데이터를 기반으로 대칭키 생성 -> 비대칭키를 통해 서버로 보내짐
