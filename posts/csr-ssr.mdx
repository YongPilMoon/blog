## CSR, SSR, Pre-rendering
<strong>CSR(Client Side Rendering)</strong>
<br />
일반적인 React 웹의 렌더링 과정과 동일합니다. 사용자가 웹에 URL을 통해 접근하면
빈 HTML과 Javascript 파일들을 다운로드 합니다. 다운로드된 Javascript에는 화면을
렌더링 하는 코드가 포함되어 있고 렌더링 및 이벤트 등록 작업이 모두 완료되면
사용자는 화면을 볼 수 있습니다. 그전까지는 빈 화면이 보입니다.

<br />
<strong>SSR(Server Side Rendering)</strong>
<br />
서버에서 렌더링 과정이 이루어집니다. 사용자가 웹에 접근하면 서버에서는 요청 페이지의
렌더링을 마친 후 레이아웃이 포함된 HTML을 내려줍니다. 이때 렌더링 시 외부 데이터가
필요하다면 data featching까지 서버에서 이루어집니다. 그로인해 사용자는 브라우저에서
Javascript가 수행되기 전에 렌더링된 화면을 먼저 볼 수 있고 그 후 필요한 Javascript가
수행되면서 각 element에 이벤트 등록이 완료되어 인터렉션이 가능해 집니다. 이 과정은
React.hydrate() 함수에 의해 이루어지는데 hydrate는 물이 다른 화합물에 결합된다는
뜻을 가지고 있습니다.
