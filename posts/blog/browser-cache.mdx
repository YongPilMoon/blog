브라우저는 HTML, CSS, Javascript, 이미지, 폰트, 비디오 등의 정적인 파일들을 캐시합니다.
<br/>

## 브라우저 캐시 동작
1.첫 번째 요청: 브라우저가 서버에 웹 페이지를 요청합니다.
<br/>
2.응답과 캐시: 서버는 웹 페이지를 응답하고, 브라우저는 캐시에 웹 페이지를 저장합니다. 파일 식별자인 ETag값을 헤더에 포함하여 보냅니다.
<br/>
3.두 번째 요청: 동일한 페이지에 대한 두 번째 요청이 발생합니다. 
<br/>
4.캐시 유효성 확인: 브라우저는 캐시에 저장된 페이지의 유효성(max-age)을 확인합니다.
<br/>
5.캐시 유효: 캐시가 유효한 경우, 캐시된 페이지를 브라우저가 바로 보여줍니다.
<br/>
6.캐시 만료: 캐시가 만료된 경우, 브라우저는 서버에 새로운 요청을 보냅니다. 이전에 저장된 ETag값을 If-None-Match 헤더에 포함합니다.
<br/>
7.If-None-Match값 확인: If-None-Match을 확인하여 요청한 파일에 변경이 있었는지 확인합니다.
<br/>
8.304 Not Modified 응답: 파일의 변경이 없다면 304 코드를 반환하고 브라우저는 기존 캐시된 파일을 사용합니다. 
<br/>
9.새로운 응답과 캐시: 파일이 변경되었다면 서버가 새로운 페이지를 응답하면, 브라우저는 이를 캐시에 저장합니다. 이후, 캐시된 페이지가 유효한 경우, 브라우저는 서버에 다시 요청하지 않고 캐시된 페이지를 보여줍니다.
