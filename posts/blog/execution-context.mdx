자바스크립트 코드가 내부적으로 어떻게 동작하는지 이해하려고 글을 썼습니다. 코드 동작 원리를 알고 있으면
자칫 햇갈릴 수 있는 호이스팅, 클로저, 스코프 체이닝등 자바스크립트 개념들을 조금 더 명확하게 이해할 수 있습니다.

<br/>
## 실행 컨텍스트(Execution Context)
자바스크립트 코드가 어떻게 실행되는지 이해하려면 먼저 자바스크립트의 핵심 개념인 **실행 컨텍스트**가 무엇인지 이해 해야 합니다.
<br/>
> 실행 컨텍스트란 스크립트나 함수가 실행 될 때 생성되는 객체 입니다.
<br/>
스크립트가 실행될때 생성되는 실행컨테스트를 `전역 컨텍스트` 함수가 실행될때 생성되는 실행컨테스트를 `함수 컨텍스트`라고 합니다.
실행컨테스트의 구조는 변수와 함수가 저장되는 메모리 영역과 코드가 실행되는 코드 영역으로 나뉩니다.
메모리 영역은 key, value 형태로 관리됩니다.
<br/>

실행 컨테스트는 코드가 종료되면 같이 삭제되고 이러한 과정은 call stack에 의해서 관리됩니다.
<br/>
자바스크립트 코드는 2단계에 걸쳐 실행이 되는데
변수, 함수가 메모리에 할당이되는 사전평가 단계, 실제 코드가 실행되는 실행단계로 나눌 수 있습니다.

첫번째 단계에서는 변수나 함수의 메모리 할당이 이루어집니다.
key에는 변수혹은 함수 이름이 들어가고 value에는 변수는 undefined 함수는 함수코드가 그대로 들어갑니다.
두번째 단계에서는 코드가 실행 됩니다. 

```
var num = 3;
function square(num) {
    return num * num;
}

var sqauredNum = sqaure(n)

console.log(sqauredNum)
```
위와 같은 코드가 실행된다고 생각해 봅시다.
