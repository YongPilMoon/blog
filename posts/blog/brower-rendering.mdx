### 1. HTML 파싱 및 DOM 트리 생성
브라우저는 HTML 파일을 로드하고, 문자열로 변환합니다.
변환된 문자열은 브라우저의 HTML 파서에 의해 파싱되어 DOM(Document Object Model) 트리를 생성합니다.
DOM 트리는 HTML 문서의 각 요소(Element)와 해당 요소의 속성(Attribute), 텍스트(텍스트 노드) 등을 노드(Node)로 포함하며, 이를 통해 JavaScript를 이용하여 HTML 문서를 동적으로 조작할 수 있습니다.
<br/>
### 2. CSS 파싱 및 CSSOM 트리 생성
브라우저는 CSS 파일을 로드하고, 문자열로 변환합니다.
변환된 문자열은 브라우저의 CSS 파서에 의해 파싱되어 CSSOM(Cascading Style Sheets Object Model)을 생성합니다.
<br/>

### 3. DOM과 CSSOM 결합 및 랜더 트리 생성
브라우저는 DOM과 CSSOM을 결합하여 랜더 트리(Render Tree)를 생성합니다.
랜더 트리는 브라우저의 뷰포트(Viewport) 영역에 표시되는 요소(Element)와 해당 요소의 스타일 정보를 포함하며, 이를 통해 브라우저는 요소의 크기와 위치, 색상, 폰트 등의 스타일을 결정합니다.
<br/>

### 4. 레이아웃(Layout) 및 페인팅(Paint)
브라우저는 랜더 트리를 이용하여 요소의 크기와 위치 등을 계산합니다.
이후, 브라우저는 계산된 정보를 이용하여 요소를 뷰포트에 배치하고, 요소에 대한 스타일을 적용합니다.
이후, 브라우저는 요소를 뷰포트에 그리고(Paint) 출력합니다.
<br/>
